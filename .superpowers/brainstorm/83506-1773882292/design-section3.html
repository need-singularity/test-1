설계 섹션 3: 진화 엔진 + 인과 추적 + 창발 감지

핵심 자율 루프의 세부 동역학

3-1. 진화 엔진 (EvolutionEngine)

세대 루프 (1회)

1. 선택 (Selection)
토너먼트 선택 (tournament_size=3)
상위 20% 엘리트 자동 보존
fitness = w₁·emergence + w₂·benchmark + w₃·efficiency

2. 인과 기반 표적 변이 (Causal Targeted Mutation)
CausalTracer가 "어떤 계층이 약점인지" 분석
→ 약점 계층만 변이 (나머지 고정)
예: "검증 계층이 환각률의 주 원인" → 검증만 교체
vs 기존 EA: 모든 계층 무작위 변이

3. 교차 (Crossover)
두 부모 후보의 계층을 교차 조합
uniform crossover (각 계층 50% 확률로 부모 선택)
인과 분석 결과 반영: 강한 계층은 보존 확률 높임

4. 다양성 보호 (Diversity Protection)
해밍 거리 기반 중복 제거
조합이 너무 수렴하면 무작위 이민자(immigrant) 주입
전체 조합 공간 대비 탐색률 추적

3-2. 인과 추적 (CausalTracer)

입력: 세대 히스토리
세대 N: {repr:A, reason:B, ...} → fitness: 0.6
세대 N+1: {repr:A, reason:C, ...} → fitness: 0.8

변경: reason B→C
결과: fitness +0.2, emergence β₁ +2
출력: 인과 어트리뷰션
인과 그래프:
reason 변경 ──→ β₁ 증가 (강도: 0.82)
reason 변경 ──→ 환각률 감소 (강도: 0.65)
reason 변경 ──✕─→ 효율 (강도: 0.03)

결론: reason 계층이 핵심 레버
인과 분석 알고리즘
방법: Ablation-based causal attribution
• 세대 간 변경된 구성요소 식별
• 각 변경이 각 지표에 미친 영향을 편미분 근사로 계산
• Δmetric_i / Δcomponent_j → 인과 강도 행렬
• 다수 세대 누적 → 통계적 유의성 검증 (p < 0.05)
비용: 추가 시뮬레이션 0회 (이미 실행된 데이터만 분석)

3-3. 창발 감지 (EmergenceDetector)

5개 지표 × 급등 판정 기준
지표 계산 방법 급등 판정 의미
β (베티수) GUDHI persistent homology |Δβ_n| > 2σ (이동 표준편차) 새 위상 구조 출현
χ (오일러 특성) Σ(-1)^n · β_n |Δχ| > 2σ 전역 위상 복잡도 변화
r (순서 매개변수) 쿠라모토 order parameter dr/dt > threshold 동기화 상전이
Φ (통합 정보) pyphi 또는 근사 계산 Φ > Φ_critical 정보 통합도 임계 초과
λ (리아프노프) 궤적 발산률 수치 추정 λ: 음→양 전환 혼돈 진입 (분기점)
급등 이벤트 발화 시 자동 동작
1. 해당 세대의 전체 상태 스냅샷 저장
2. CausalTracer에 즉시 인과 분석 요청
3. emergence_events.jsonl에 이벤트 기록
4. claude CLI 호출 → 급등 분석 리포트 자동 생성
5. 해당 조합을 hall_of_fame에 등록
